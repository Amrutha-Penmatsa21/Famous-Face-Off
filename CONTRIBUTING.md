# Contributing to CLARA

Thank you for your interest in contributing! This document provides guidelines for contributing to the Famous Face-Off project.

## Code of Conduct

- Be respectful and inclusive
- Provide constructive feedback
- Follow the existing code style

## How to Contribute

### 1. Report Bugs
- Use GitHub Issues to report bugs
- Include steps to reproduce
- Provide system information (Python version, OS, etc.)

### 2. Suggest Enhancements
- Check existing issues first
- Be clear and descriptive
- Propose implementation approach if possible

### 3. Submit Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/YourFeature`)
3. Make your changes
4. Commit with clear messages (`git commit -m 'Add your feature'`)
5. Push to your fork (`git push origin feature/YourFeature`)
6. Open a Pull Request with description

## Code Style

- Follow PEP 8 conventions
- Use meaningful variable names
- Add comments for complex logic
- Keep functions focused and small

## Testing

- Test your changes thoroughly
- Include example images for testing
- Test on multiple Python versions if possible

## Documentation

- Update README.md if adding features
- Add docstrings to functions
- Include comments for complex algorithms

## Questions?

Create an issue with a question tag or contact the maintainers.

---

Thank you for contributing! 🙏
